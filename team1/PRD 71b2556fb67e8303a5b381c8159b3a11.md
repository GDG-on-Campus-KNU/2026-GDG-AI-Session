# PRD

1. A개요 (목적과 배경)
    - 배경 : 현재 학과 및 동아리의 물품 대여는 수기 장부 또는 단체 채팅방을 중심으로 비체계적으로 관리되고 있다. 이로 인해 물품의 실시간 재고 현황을 정확하게 파악하기 어렵고, 대여 및 반납 이력 관리가 체계적으로 이루어지지 않는 문제가 발생하고 있다. 특히 반납 지연이나 분실 상황 발생 시 책임 소재를 추적하기 어렵고, 관리자의 운영 부담이 증가하는 한계가 존재한다.
    - 기존 수기 장부나 메신저(단톡방)에 의존하던 비효율적인 관리 방식을 디지털화하여, 실시간 재고 추적과 대여 프로세스의 투명성을 확보하는 것을 목적으로 한다.
    
    ---
    
2. 목표
    - 물품 관리자의 재고 관리 접근성 개선(분실률 10% 감소)
    - 물품 대여자의 대여 편의성 개선(대여 서비스 사용률 10% 증가)
    - 대여/반납 프로세스 시간 단축(행정에 드는 시간 70% 감소)
    - 대여 시스템 신뢰성 향상(대여 프로세스 과정에서 발생하는 문제 발생 건수 0회)

1. 사용자 스토리 or 페르소나
    - 관리자 (학생회/조교)
        - 페르소나 : 교내 연합 동아리 물품 관리자, 관리 품목으로는 행사용 앰프/ 마이크, 보드게임 등
        - 주요 고충으로 축제시즌이 되면 대여/반납 요청이 폭주함. 대여 현장에서 일일이 학생증을 확인하고 장부에 적을 시간이 없음. 연체하는 학생들이 많지만, 일일이 카톡을 보내 독촉하기가 껄끄럽고 번거로움.
        - 시나리오
            1. 관리자가 학생이 제시한 ‘예약 내역 QR코드를 스캔’ → 대여 승인 페이지로 즉시 이동
            2. 이전에는 수기 장부에서 학생 정보를 찾고, 재고를 일일이 대조하며 서명을 받아야 해 번거롭고 대기열이 길었음.
            3. 개선 후, 대여 승인 페이지에서 예약자 신원과 불출해야할 물품 리스트를 한눈에 확인 가능
            4. 승인 화면에서 물품 누락이나 파손이 없는지 체크한 뒤, ‘대여 승인’ 버튼을 눌러 즉시 불출 진행
            5. 관리자는 10초만에 대여 처리를 완료, 대여 완료 화면에서 곧바로 대기자의 물품을 스캔할 준비를 마침
    - 물품 사용자 (학생) :
        - 유령 물품 상황 ) 공용 물품 보관함에 있어야 할 물품이 보이지 않는 상황 / 반납 완료라고 써있지만 실제로는 보이지 않는 상황 - 추가 발생 피해로 불필요한 지출 - 나는 아니겠지 하는 마인드 - 실질적인 최종위치와 현재 점유 사용자 가 있어, 실질적인 강제성 부여
        - 반납 상황의 불연속성과 헛걸음 하는 상황 ) A 는 물건을 빌리고, 시험기간과 공강 시간에 큰 마음 먹고 반납하러 갔지만, 문이 닫겨 있어, 반납을 못하고 돌아가는 상황 . - 그렇게 2주정도 해당 물건을 반납하지 못하고, 돌아 가는 상황. “분명히 반납 하려 했어” 의 자기 합리화로, 다음에 하자고 하며 반납을 하지 못함. - 반납 포기 상황 (심리적 장벽) - 사용자가 반납의지 가 있을 때, 기본적인 반납 정보를 기록하는 QR 서비스 필요.

2. 핵심 기능 요구사항
    - 회원 및 인증 : 회원가입, 로그인, 프로필
        - 회원가입/로그인: 학교 이메일 인증 절차 완료 후 로그인/가입 요청 시 POST /api/v1/auth/register (또는 login) API를 호출하며, 성공 응답으로 반환된 JWT를 로컬에 저장하여 이후 통신에 활용한다.
        - 프로필: 마이페이지 진입 시 GET /api/v1/users/me API를 호출하며, 성공 응답 시 사용자의 기본 정보와 페널티 만료일(penalty_end_date) 데이터를 화면에 바인딩한다.
    - 물품 조회 및 검색 : 전체 목록 보기, 카테고리별 필터링, 물품 상세 정보
        - 전체 목록 보기: 메인 화면 진입 시 GET /api/v1/items API를 호출하며, 응답받은 페이지네이션 기반의 전체 물품 리스트 데이터를 목록 UI에 렌더링한다.
        - 카테고리별 필터링: 특정 카테고리 탭 선택 시 카테고리 식별자를 파라미터로 포함하여 GET /api/v1/items API를 재호출하며, 성공 응답 시 해당 카테고리의 물품들로만 리스트 뷰를 갱신한다.
        - 물품 상세 정보: 리스트 내 특정 물품 클릭 시 해당 물품 ID를 포함하여 API를 호출하며, 성공 응답 시 물품의 메타데이터와 개별 기기 기반의 실시간 대여 가능 수량을 표시한다
    - 예약 및 대여 : 실시간 예약, 중복 방지, 예약 취소
        - 실시간 예약: 희망 대여 기간을 선택하고 예약 버튼 클릭 시 POST /api/v1/reservations API를 호출하며, 성공 응답 시 예약 상태를 확정하고 완료 화면으로 라우팅한다.
        - 중복 방지: 실시간 예약 API 호출 시 백엔드의 Redis 분산 락에 의해 동시성 제어가 수행되며, 재고 소진으로 인한 에러 응답 수신 시 클라이언트는 사용자에게 예약 불가 알림 다이얼로그를 노출한다.
        - 예약 취소: 사용자가 대여 승인 전 취소 버튼 클릭 시 PATCH /api/v1/reservations/{reservationId}/cancel API를 호출하며, 성공 응답 시 로컬 화면의 예약 상태를 CANCELED로 변경하고 UI를 새로고침한다.
    - 관리자 제어 : 물품 관리, 대여 승인, 반납 처리
        - 물품 관리: 관리자가 신규 물품 등록 시 POST /api/v1/admin/items API를 호출하여 다건(Bulk) 등록을 수행하고, 개별 기기 파손/분실 처리 시 PATCH /api/v1/admin/item-units/{unitId}/status API를 호출하여 해당 기기의 상태를 변경한다.
        - 대여 승인: 관리자가 현장에서 사용자 예약 건의 QR/바코드를 스캔하여 페이로드를 검증한 뒤 PATCH /api/v1/admin/reservations/{reservationId}/approve API를 호출하며, 성공 시 해당 예약 상태를 RESERVED에서 RENTED로 전이시킨다.
        - 반납 처리: 관리자가 회수한 물품의 QR/바코드를 스캔하여 PATCH /api/v1/admin/reservations/{reservationId}/return API를 호출하며, 성공 시 해당 예약 상태를 RENTED에서 RETURNED로 변경하고 가용 재고를 복구한다.
    - 알림 및 이력 : 자동 알림, 이력 관리
        - 자동 알림: 백엔드의 배치 스케줄러가 반납 임박/연체 상태(OVERDUE)를 감지하여 FCM으로 푸시를 발송하면, 클라이언트는 이를 수신하여 시스템 알림을 띄우고 터치 시 해당 예약 상세 화면으로 이동시킨다.
        - 이력 관리: 내 대여 이력 화면 진입 시 GET /api/v1/reservations/me API를 호출하며, 성공 응답 시 반환된 데이터를 바탕으로 사용자의 과거 대여 기록과 현재 대여/예약 진행 목록을 출력한다.

3. 제외범위
    - 외부 포털(학교 계정 등) 시스템과의 연동은 제외한다.
    - 벌점이나 페널티 등 점수 기반 신뢰도 시스템은 이번 버전에서 제외한다.
    - 과도한 UX/UI 확장을 제외한다.
        - 커뮤니티 기능
        - 게이미피케이션 기능
    - 고도화된 추천 및 개인화 서비스는 제외한다.
        - 추천 기능
    - 복잡한 예약 정책 및 자동화 로직은 제외한다.
        - 대기열 자동 승격
        - 시간대별 세밀한 예약 슬롯 관리

4. 성공지표 
    - 시스템 안정성: 트래픽이 몰리는 축제/시험 기간의 API 답 속도 평균 1초 이내 유지, 인기 물품 예약 충돌(overbooking) 발생 0건
    - 운영 안정성: 기존 수기 관리 대비 대여/반납 처리 소요시간 n%감소
    - 활성도: 서비스 이용후 2주 내 재이용 비율 n%
    - 연체 관리: 시스템 도입 이전 대비 평균 연체율 30%이상 감소
