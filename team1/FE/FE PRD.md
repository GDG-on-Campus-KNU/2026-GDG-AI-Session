## UX

1. **시각적 피드백과 상태공유**
    
    > 사용자가 자신이 누른 버튼이 작동하는지를 즉각적 피드백
    > 
    - Action: 예약 버튼을 눌렀을 때 백엔드 응답을 기다리는 동안 버튼을 비활성화하고 로딩 스피너를 보여주어 중복 클릭 방지
    - Action: 대여 성공, 실패(재고부족), 연체 경고 등은 명확한 색상(성공: 초록, error: 노랑, 경고: 빨강)의 Toast 메시지
2. **체감 성능 향상 (사용자가 느끼는 응답속도)**
    
    > 실제 api 속도를 사용자가 빠르다고 느끼도록
    > 
    - **Action:** 물품 목록을 불러올 때 빈 화면이나 단순한 빙글빙글 로딩 대신, **Skeleton UI(뼈대 화면)**를 보여주어 콘텐츠가 곧 뜬다는 심리적 안정감을 줍니다.
    - **Action:** React의 상태 관리를 활용해, '좋아요(관심 물품)' 같은 가벼운 기능은 서버 응답을 기다리기 전에 화면에서 먼저 하트를 채워주는 **Optimistic UI(낙관적 업데이트)**를 적용해 볼 수 있습니다.
3. **예외 상황에서의 부드러운 대처** 
    
    > 동시 예약에 의한 실패에 대한 대처
    > 
    - **Action:** 내가 예약을 시도했는데 0.1초 차이로 다른 사람이 먼저 가져가서 에러가 났을 때, 단순한 "에러 발생" 팝업이 아니라 “방금 다른 분이 마지막 물품을 대여했어요. 다른 물품을 찾아보시겠어요?"라는 안내 메시지와 함께 **자동으로 물품 목록을 새로고침(Refetch)** 해주는 흐름
4. **모바일 최우선 사용성 (Mobile-First Usability)**
    
    > 사용자 특성 상 대학교 내에서 휴대폰을 접근하는 비율이 높음.
    > 
    - **Action:** 주요 액션 버튼(예약하기, 반납 바코드 띄우기)은 화면 하단(Thumb zone)에 크고 터치하기 쉽게 배치합니다.
    - **Action:** 키보드가 올라올 때 중요한 입력창이나 버튼이 가려지지 않도록 스크롤 처리를 꼼꼼히 합니다.

## 핵심 화면 플로우

본 서비스에서 사용자는 로그인 이후 물품을 탐색하고, 예약을 신청하며, 이후 대여 상태를 지속적으로 확인하는 흐름으로 서비스를 이용한다. 

<aside>
▶️

전체 flow
로그인/회원가입 → 물품 탐색 → 물품 상세 확인 → 예약 신청 → 예약 상태 확인 → 대여 이력 관리로 이어진다.

</aside>

이 화면에서는 검색을 통해 원하는 물품을 찾거나 카테고리 필터를 통해 물품을 분류하여 탐색할 수 있으며, 각 물품 카드에서 대여 가능 여부와 남은 수량을 직관적으로 확인할 수 있다. 

사용자는 이 단계에서 현재 자신이 빌릴 수 있는 물품을 빠르게 파악하게 된다.

관심 있는 물품을 선택하면 상세 페이지로 이동하며, 이 화면에서는 물품 사진을 통해 실제 형태를 확인하고 물품명, 설명, 규격 등의 정보를 확인할 수 있다. 또한 현재 남은 수량과 최대 대여 가능 기간이 함께 제공되며, 대여가 불가능한 경우에는 그 이유가 명확하게 안내된다. 

사용자는 이 단계에서 해당 물품을 지금 빌릴 수 있는지와 언제까지 사용할 수 있는지를 판단하게 된다.

사용자가 대여를 결정하면 예약 신청 화면으로 이동한다. 이 화면에서는 대여 시작일과 반납 예정일을 선택하고, 필요한 경우 수량을 지정하며, 예약이 불가능한 날짜나 조건에 대한 안내를 사전에 확인할 수 있다. 

모든 조건을 확인한 뒤 예약을 신청하면 요청이 정상적으로 접수된다. 이 단계는 단순한 화면 이동이 아니라 실제 대여 요청이 생성되는 핵심 단계이다.

예약이 완료되면 사용자는 예약 결과를 확인할 수 있으며, 예약 성공 여부와 함께 현재 상태가 명확하게 표시된다. 

상태는 승인 대기, 승인 완료, 대여 중, 반납 완료, 연체 등으로 구분되며, 사용자는 이를 통해 자신의 예약이 어떤 단계에 있는지를 직관적으로 파악할 수 있다. 이 정보는 이후 실제 대여 및 반납 과정까지 이어진다.

마지막으로 사용자는 내 대여 내역 화면에서 현재 및 과거 대여 정보를 확인할 수 있다. 이 화면에서는 현재 대여 중인 물품과 반납 예정일, 연체 여부를 확인할 수 있으며, 과거 대여 이력 또한 함께 제공된다. 

또한 승인 전 상태의 예약에 대해서는 취소가 가능하다. 이를 통해 사용자는 자신이 빌린 물품의 상태를 지속적으로 확인하고 관리할 수 있다.

1. 사용자 핵심 기능 플로우 : 로그인/회원가입 → 홈(물품 목록) 
→ 물품 상세 → 예약 신청 → 예약 완료/상태 확인 → 내 대여 내역

1. 로그인 / 회원가입 화면

사용자는 서비스에 처음 진입하면 학교 이메일을 기반으로 인증을 진행한다. 이 과정은 최대한 간단하게 설계되어 이메일 입력과 인증 코드 확인만으로 로그인 또는 회원가입이 동시에 처리된다. 

최초 가입 시에는 이름, 학번, 학과 등의 기본 정보를 입력하며, 이후에는 별도의 절차 없이 바로 서비스 이용이 가능하다. 이를 통해 사용자는 복잡한 가입 과정 없이 빠르게 서비스에 진입할 수 있다.

로그인 이후 사용자는 홈 화면에서 현재 대여 가능한 물품들을 확인할 수 있다.

 

1. 이메일 입력
2. 인증 코드 입력
3. 로그인 / 회원가입 분기 최소화
4. 최초 가입 시 기본 정보 입력(이름, 학번, 학과)
5. 홈(물품 목록)
    1. 검색창
    2. 카테고리 필터
    3. 물품 카드 리스트
    4. 대여 가능 여부 / 남은 수량 표시
    5. 인기 물품이 아니라도 우선은 전체 목록 중심
6. 물품 상세
    1. 물품 사진
    2. 물품명
    3. 설명 / 규격
    4. 남은 수량
    5. 최대 대여 가능 기간
    6. 예약 버튼
    7. 대여 불가 시 상태 안내
