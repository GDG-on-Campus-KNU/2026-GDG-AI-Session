# PRD
  ## 1. 문제정의
        현재 대학 내 공용 물품 대여는 수기 장부, 단톡방, 구두 전달 등 비정형적인 방식으로 운영되고 있다. 이로 인해 다음과 같은 문제가 발생한다:
        
        - 사용자 관점 (학생)
            - 대여 가능 여부를 사전에 확인할 수 없어 불필요한 이동 발생
            - 예약 여부가 불명확하여 중복 요청 및 혼선 발생
            - 반납 일정에 대한 관리가 어려워 연체 발생
        - 관리자 관점 (학생회/조교)
            - 물품 재고 및 상태를 실시간으로 파악하기 어려움
            - 반납 지연 시 추적 및 관리에 많은 시간 소요
            - 예약 요청이 분산되어 승인 프로세스 비효율 발생

## 2. 사용자 정의 (User Personas)


| 사용자 구분 | 설명 | 주요 역할 |
| :--- | :--- | :--- |
| **일반 사용자 (학생)** | 서비스를 통해 물품을 빌리는 실사용자 | 물품 검색, 예약 신청, 대여 현황 확인, 반납 알림 수신 |
| **관리자 (학생회/조교)** | 물품과 대여 상태를 총괄하는 운영 주체 | 물품 등록/수정, 예약 승인/거절, 반납 확인, 연체자 관리 |
 ## 3. 프로젝트 목표 (Goals)


### 👤 학생 (일반 사용자)
* **대여 가능 여부 실시간 확인:** 직접 방문하지 않고도 물품의 재고 상태를 사전에 파악하여 불필요한 이동을 방지한다.
* **편리한 예약 및 상태 추적:** 원하는 기간에 물품을 예약하고, 내 대여 현황(대여 중/승인 대기 등)을 실시간으로 확인한다.
* **명확한 반납 안내:** 반납 일정과 상태를 체계적으로 안내받아 의도치 않은 연체를 방지한다.

### 🛠 관리자 (학생회/조교)
* **통합 재고 현황 대시보드:** 물품의 재고 상태와 전체 대여 현황을 한눈에 파악하여 관리 공수를 줄인다.
* **승인 프로세스 단일화:** 분산된 예약 요청을 하나의 시스템으로 통합하여 승인 및 반려 절차를 단순화한다.
* **체계적인 연체 및 이력 관리:** 연체자 식별과 물품 상태 기록을 디지털화하여 관리의 투명성과 회수율을 높인다.

## 5. User Flow
        

        학생
        → 물품 검색
        → 상세 확인
        → 예약 신청
        → 관리자 승인
        → 물품 수령
        → 반납

        

        관리자
        
        → 예약 요청 확인
        → 대여 승인
        → 물품 전달
        → 대여 상태 변경
        → 반납 확인
        → 반납 완료 처리
        
## 6. 제외 범위 (Out of Scope)


| 항목 | 제외 사유 |
| :--- | :--- |
| **결제 시스템** (보증금·연체료 온라인 결제) | 초기에는 오프라인 처리로 충분하며, 결제 연동 시 별도의 금융 인프라 구축이 필요함. |
| **카카오 알림톡 연동** | 내부 앱 푸시 알림으로 1차 대응하며, 외부 채널 연동은 차기 업데이트로 이관함. |
| **QR코드 기반 자동 대여·반납** | 안드로이드 앱 1차 배포 및 안정화 이후 도입 여부를 검토함. |
| **물품 추천 기능** | 추천 알고리즘 적용을 위한 충분한 대여 이력 데이터 축적 이후 도입함. |
| **다기관·타학과 간 물품 공유** | 단일 기관(학과/학생회) 운영을 우선 검증한 후 확장 여부를 결정함. |
| **커뮤니티 기능** | 사용자 간 소통보다는 1:1 예약 확인 및 안내 알림 기능에 우선 집중함. |
| **일회용품 대여** | 일회용품은 반납 프로세스가 모호하므로 향후 별도 논의 후 추가 여부 결정. |

## 7. 성공 지표 (Success Metrics)


| 지표명 | 목표 수치 | 측정 기준 및 의미 |
| :--- | :--- | :--- |
| **물품 목록 조회 응답 시간** | **p99 기준 1초 이내** | 대량의 물품 데이터 조회 시에도 쾌적한 사용자 경험 유지 |
| **예약 프로세스 완료율** | **80% 이상** | 예약 신청 시작 대비 이탈 없이 완료까지 도달하는 비율 |
| **연체율** | **5% 미만** | 자동 알림 시스템을 통한 정기적인 반납 유도 효과 측정 |
| **동시 예약 정합성 오류** | **0건** | 동일 물품에 대한 중복 예약 발생 차단 (데이터 무결성) |
| **관리자 일일 처리 시간** | **30분 이내** | 수기 장부 대비 대여/반납 승인 및 관리 업무 효율화 |
