# 1. 배경 (Background / Context)

학과 및 동아리에서 사용하는 디지털 기기나 행사 용품은 대부분 수기 장부나 단체 카카오톡 방을 통해 대여가 관리되어 왔습니다. 이러한 방식은 다음과 같은 페인 포인트를 가지고 있습니다:

- **재고 파악의 어려움:** 현재 남은 수량을 실시간으로 확인할 수 없어 예약이 중복되는 경우가 많음.  
- **반납 추적 불가능:** 담당자가 반납 지연을 일일이 체크해야 하며, 지연된 물품을 찾기 어려워 분실 위험이 존재함.  
- **관리 리소스 낭비:** 수기로 일정을 관리하는 데 많은 시간이 소요되고, 자원 활용도가 떨어짐.  

KNU-Share 서비스는 이러한 비효율을 해결하고, 학내 공용 물품의 투명한 사용과 효율적인 운영을 위해 기획되었습니다. 최신 연구에서는 디지털 자산 관리 시스템이 자원 이용률을 평균 20% 이상 증가시키고, 관리 시간을 40% 이상 단축하는 것으로 보고되고 있습니다.

---

# 2. 목표 (Goals)

- **예약 관리 자동화:** 대여 신청과 승인/반납 과정을 디지털화하여 수작업 절차를 최소화한다.  
- **실시간 재고 및 상태 표시:** 모든 사용자에게 실시간으로 남은 수량과 상태(Available → Reserved → Rented → Returned/Overdue)를 보여준다.  
- **대여 지연 감소:** 푸시 알림과 연체 페널티 정책을 통해 반납 지연율을 30% 이상 감소시키는 것을 목표로 한다.  
- **사용자 만족도 향상:** 편리한 검색 및 예약 경험을 제공하여 대여 서비스 만족도를 80점 이상(100점 만점 기준)으로 향상시킨다.

# 7. 예외 케이스 및 FAQ (Edge Cases)

| 시나리오 | 대응 정책 |
|----|----|
| 동시 예약 경쟁 | 인기 물품(예: 아이패드)에 대해 여러 사용자가 동시에 예약을 시도할 경우 데이터베이스 락 또는 Redis 분산락을 사용해 정합성을 보장한다. 선착순 기준으로 대여자를 확정하고 추가 요청자는 대기자로 처리한다. |
| 승인 후 예약 취소 요청 | 사용자가 대여 승인을 받은 뒤 예약 취소를 요청할 경우 관리자에게 문의하도록 안내한다. 반납 예정일 이전 취소 시 연체 페널티 없이 처리할 수 있다. |
| 반납 연체 | 반납 예정일시 이후에도 상태가 ‘Rented’인 경우를 자동으로 ‘Overdue’로 변경하고, 매시간 배치 작업에서 연체 여부를 확인해 알림을 발송한다. 연체 1일당 7일간 대여 금지라는 페널티를 적용한다. |
| 네트워크 장애 | 결제 기능이 없으므로 네트워크 오류로 예약 프로세스가 중단될 경우, 브라우저/앱에서 재시도 옵션을 제공한다. 서버 측에서 상태 머신에 따라 일시적인 Pending 상태로 유지하고 일정 시간 후 사용자가 다시 시도할 수 있도록 한다. |
| 프로필 정보 변경 중 세션 만료 | 사용자가 프로필 수정 도중 세션이 만료되면 수정 내용이 서버에 저장되지 않는다. 세션 만료 시 로그인 화면으로 리다이렉션하고, 이전에 입력한 내용은 클라이언트 캐시에 임시 저장하여 재로그인 후 자동으로 복구할 수 있도록 한다. |
| 관리자 실수로 반납처리를 누락 | 시스템은 예약 시점과 예정 반납일을 기반으로 자동으로 연체 상태를 업데이트하므로, 관리자의 버튼 입력이 늦어도 연체 정보는 유실되지 않는다. |
| 대여 한도 초과 | 한 사용자가 동시에 빌릴 수 있는 물품 종류와 개수를 관리자가 설정할 수 있도록 하며, 초과 시 예약 버튼을 비활성화하고 사유를 알린다. |

# 8. 기술 설계 개요 (Technical Design Summary)

- **동시성 처리:** 인기 물품에 대한 동시 예약을 처리하기 위해 DB transaction isolation level을 적절히 설정하고, 필요한 경우 Redis 기반 분산락을 적용한다.
- **상태 머신:** 물품의 상태는 Available → Reserved → Rented → Returned/Overdue 순으로 흐르며, 각 전환에는 관리자 승인이나 자동 스케줄러가 개입한다. 상태 변경은 원자적 트랜잭션으로 처리한다.
- **배치 작업:** 매시간 반납 예정일을 확인하여 연체된 물품을 Overdue로 표시하고, Overdue 상태의 사용자에게 알림을 발송한다.
- **실시간 업데이트:** 프론트엔드와 안드로이드 클라이언트는 WebSocket 또는 Server-Sent Events를 사용해 물품 상세 페이지의 잔여 수량을 실시간으로 반영한다.
- **UX 가이드:** 대여/반납 과정은  
  1) 물품 선택 →  
  2) 예약 정보 입력 →  
  3) 예약 확인 →  
  4) 대여 승인(관리자) →  
  5) 반납 완료  
  의 단계로 시각화한다. 모바일 사용성을 고려해 주요 기능을 최소한의 터치로 완료할 수 있게 설계한다.

# 9. 비기능 요구사항 (Non-Functional Requirements)

- **성능:** 물품 검색 및 목록 조회의 서버 응답 시간은 1초 이내를 목표로 한다. 동시 접속량(RPS)은 축제 기간을 고려해 사전에 부하 테스트를 수행해 설정한다.
- **안정성:** 관리자가 반납 버튼을 누르지 않더라도 시스템이 자동으로 연체 상태를 파악해 데이터가 유실되지 않도록 설계한다.
- **가용성:** 축제 등 대여 수요가 급증하는 시기에도 서비스가 중단되지 않도록, 클라우드 인프라의 Auto-scaling 및 무중단 배포 정책을 적용한다.
- **보안:** 학교 이메일 인증 외에도 토큰 기반 인증(JWT)을 사용해 세션을 관리하고, 개인정보는 암호화하여 저장한다. 주요 API는 HTTPS로만 통신한다.

# 10. 향후 과제 (Roadmap)

- 바코드/QR코드 스캔 기능: 모바일 앱에 장착된 카메라를 이용해 물품을 스캔하고 신속하게 대여/반납을 확인하는 기능을 개발한다.
- 외부 시스템 연동: KNU 이외의 타 대학 물품 공유 서비스와의 연동 및 통합 검색을 검토한다.
- 사용자 리뷰 및 평가 기능: 물품 상태나 만족도를 평가할 수 있는 리뷰 시스템을 추가해 품질 개선에 활용한다.
