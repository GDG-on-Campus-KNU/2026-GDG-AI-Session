# 📌 Picnic Rental Service PRD (Frontend)

---

## 1. 배경 (Background)

봄 시즌 도래로 피크닉 수요 증가  
→ 학생들이 돗자리, 담요 등 물품을 사용하고 싶어함  
→ 하지만 현재는 **직접 가서 확인해야만 재고 파악 가능**

👉 문제:
- 재고 확인 불편
- 대여 접근성 낮음
- 타 학과 물품 이용 어려움

👉 해결:
- 앱 기반 물품 대여 서비스 제공
- 직관적인 재고 확인 + 예약 시스템 구축
- 향후 유료 서비스 확장 가능

---

## 2. 목표 (Goal)

- 경북대학교 학생들의 **편리한 문화생활 지원**
- 학과 간 **물품 공유 및 교류 활성화**
- 간편한 예약 경험 제공

---

## 3. 유저스토리 & 페르소나

### Persona A
- 컴퓨터학부 학생
- 타과 친구들과 피크닉 계획

👉 Pain Point  
- 특정 학과 물품 대여 접근이 애매함
- 재고 확인이 번거로움

👉 Needs  
- 한 번에 확인 가능한 시스템
- 간편한 예약 UX

---

### Persona B
- 비주기적으로 피크닉을 즐기는 일반 학생

👉 Pain Point  
- 어떤 물품이 있는지 모름
- 대여 방법이 불명확함

👉 Needs  
- 추천 기반 UI
- 직관적인 상태 표시

---

## 4. 핵심 기능 요구사항 (Frontend Scope)

### 4.1 회원 및 인증
- 로그인 / 회원가입
- 사용자 기본 정보 저장

---

### 4.2 물품 조회
- 메인 화면에서 물품 리스트 확인
- 상태를 색상으로 표현

| 상태 | 의미 |
|------|------|
| 🟢 | 대여 가능 |
| 🟠 | 일부 가능 |
| 🔴 | 대여 불가 |

- 남은 대여 시간 시각화 (Progress / 타이머)

---

### 4.3 예약 및 대여
- 물품 클릭 시 상세 정보 확인
- 대여 가능 수량 표시
- 대여 시간 선택
- 최대 대여 시간 제한

👉 예약 플로우
1. 물품 선택
2. 시간 선택
3. 예약 버튼 클릭
4. 모달로 예약 완료 표시
5. 메인으로 복귀

---

### 4.4 네비게이션 구조

하단 네브바 구성:

- 메인
- 예약내역
- 도움

---

### 4.5 예약 내역
- 현재 예약 상태 표시
- 과거 이력 확인
- 대여 시간 및 반납 시간 표시

---

### 4.6 도움 (Help)
- 반납 불가 상황 전달

예시:
- 파손
- 훼손
- 도난
- 반납 지연

👉 사용자 입력 기반 메시지 전달 기능

---

### 4.7 알림
- 푸시 알림 제공

알림 유형:
- 반납 시간 임박
- 예약 완료
- 연체 발생

---

### 4.8 관리자 페이지 (Frontend View)

관리자 로그인 후 접근 가능

기능:
- 물품별 예약 현황 확인
- 사용자 리스트 조회
- 미반납 사용자 확인

포함 정보:
- 학번
- 연락처
- 대여 상태

- 도움 메시지 확인 및 사용자 대응

## 9. 예외 케이스 (Edge Cases)

### 9.1 미반납
- 일정 시간 초과 시 알림
- 관리자 페이지에 표시

---

### 9.2 물품 훼손
- 도움 기능을 통한 신고
- 관리자 확인 후 조치

---

### 9.3 도난

- 관리자 대응 프로세스 필요

---

## 10. UX 핵심 포인트

- 재고 상태를 **즉시 인지 가능**
- 예약 과정은 **3단계 이내**
- 감성 기반 UI (피크닉 경험 강조)
- 오류 방지 중심 UX (선택 불가 처리)
